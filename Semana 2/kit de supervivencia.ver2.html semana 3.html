1. ¿Python no responde? (El problema del PATH)

Si escribes python en tu terminal y no pasa nada, o se abre la Microsoft Store, no tienes configurado el PATH. Este es el motivo #1 de bloqueo en la primera semana.

⚠️
Síntoma: Error 'python' no se reconoce como un comando interno o externo...
🤖 Prompt de Rescate
Actúa como soporte técnico de Windows/Mac. Escribo 'python' en mi terminal y recibo un error de que no se reconoce el comando. Ayúdame a: Verificar si tengo Python instalado. Agregar Python a las Variables de Entorno (PATH) paso a paso. Verificar que funcione. Mi sistema operativo es: [Windows 11 / MacOS / Linux]
📂
2. No programes todo en un solo archivo

Un error común es tener un archivo gigante. Organiza tu proyecto así desde el día 1 para evitar caos:

proyecto_ecomarket/ ├── venv/ # Entorno virtual (¡NO LO TOQUES!) ├── db.json # Base de datos del Mock Server ├── src/ # Tu código fuente │ ├── cliente_sync.py # Semana 2 (Requests) │ ├── cliente_async.py # Semana 3 (Aiohttp) │ ├── validadores.py # Funciones reutilizables │ └── config.py # URLs y constantes └── tests/ # Tus pruebas ├── test_sync.py └── test_async.py

3. Zona de Peligro: Síncrono vs Asíncrono
☠️
REGLA DE ORO: Nunca mezcles requests dentro de una función async def. Esto bloquea todo el programa y anula la ventaja de la asincronía.
Característica Semana 2 (requests) Semana 3 (aiohttp)
Tipo Bloqueante (Una cosa a la vez) No bloqueante (Multitarea)
Instalación pip install requests pip install aiohttp
Definición def obtener(): async def obtener():
Llamada requests.get(url) await session.get(url)
Cuándo usar Scripts simples, CLI, pruebas Dashboards, Scrapers, Alto rendimiento

5. Diccionario de Errores "Googleables"

Traducción de lo que Python te grita a español humano.

El Error (Python) La Realidad (Español) Solución Rápida
requests.exceptions.Timeout / asyncio.TimeoutError "El servidor tardó demasiado en contestar" 1. Siempre pon timeout explícito (ej: 10-15 s)
2. En async: async with timeout(10): o session.get(..., timeout=10)
3. Decide si reintentar (5xx sí, 4xx no)
JSONDecodeError: Expecting value... "El servidor me devolvió algo que no es JSON (probablemente HTML de error o texto vacío)". Imprime response.text antes de hacer .json() para ver qué llegó realmente.
ClientConnectorError: Connection refused "Toqué el timbre y nadie abrió". El servidor (json-server) está apagado o el puerto es incorrecto.
RuntimeWarning: coroutine ... was never awaited "Preparaste la tarea pero nunca le diste 'Start'". Te faltó poner await antes de llamar a la función async.
401 Unauthorized "No tienes permiso". Te falta el header Authorization o el token venció.
429 Too Many Requests "¡Cálmate! Vas muy rápido". Estás saturando el servidor. Usa un semáforo o duerme entre peticiones.
☠️
Regla de supervivencia #1 en aiohttp:
SIEMPRE usa async with aiohttp.ClientSession() as session:
o cierra manualmente con await session.close()

Si no lo haces → tu programa se queda sin conexiones después de 100-300 peticiones.

6. ¿Demasiadas peticiones a la vez? (429 Too Many Requests)

asyncio te permite lanzar 50, 100 o más peticiones simultáneas… pero el servidor (incluso json-server) puede colapsar o devolver 429.

⚠️
Síntomas comunes:
Muchos 429, 500, o simplemente el servidor deja de responder.
El programa parece "colgarse" aunque no hay error visible.

Solución recomendada: Usa un Semáforo

# Limita a máximo 8-12 peticiones simultáneas sem = asyncio.Semaphore(10) async def peticion_segura(url, session): async with sem: # Solo 10 corutinas entran aquí a la vez async with session.get(url, timeout=15) as resp: if resp.status != 200: raise Exception(f"Error {resp.status}") return await resp.json()

Coloca el Semaphore al inicio de tu script principal y envuelve cada petición en async with sem:.

🤖 Prompt de rescate para saturación
Estoy haciendo muchas peticiones simultáneas con aiohttp y recibo errores 429 o el servidor se cuelga. Genera un ejemplo completo usando asyncio.Semaphore para limitar a 8-12 peticiones concurrentes. Incluye manejo de timeout y reintentos simples para 5xx. Mi código actual usa asyncio.gather(...)
